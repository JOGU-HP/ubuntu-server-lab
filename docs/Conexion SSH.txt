Conexion por SSH al servidor en una misma red local

Objetivo
Conectar por cliente ssh un equipo que entrara con un usuario, reconocer el sistema y familiarizarse con la terminal y comandos

**Alcance
Utilizar comandos para configuraciones en archivos de conexion a internet
Crear directorios y archivos con visualizacion
Creacion de grupos y asignacion de usuarios a cada grupo
Instalacion de herramientas para un mejor reconocimiento de usuarios en grupos

-Pasos de configuracion-

**Instalacion de ssh en Windows 11 (maquina cliente)
Se realizo la instalacion del cliente ssh en la maquina cliente que tiene un sistema operativo Windows 11, buscando informacion de sitios web hasta videotutoriales en internet para la correcta instalacion del cliente [images/conexionssh/Instalacion cliente SSH]

**Reconfiguracion de IP
Como se trabajara en un entorno aislado, se realizo la conexion de Ubuntu Server a un router sin internet para obtener un punto de conexion entre 2 maquinas (Ubuntu Server, Windows 11, Ubuntu, Kali Linux)
Esto mantiene la seguridad de la red y evita problemas con redes ajenas a los propositos de las practicas a realizar
Los archivos modificados fueron netplan y dhcp.conf 
Se realizo un ping entre ambas maquinas con conexion al mismo punto y la comunicacion entre ambas maquinas fue satisfactoria [images/conexionssh/ping maquina cliente-servidor]

**Acceso desde Windows 11 a Servidor
Cuando la instalacion del cliente ssh en windows 11 (maquina cliente) fue satisfactoria, se procedieron a realizar pruebas de conexion a traves del cliente ssh, comprobando que los servicios DHCP y SSH esten activos y funcionales [images/conexionssh/servicios ejecutados]

Despues de corroborar que los servicios se esten ejecutando, se realiza una conexion con ssh utilizando el comando ssh usuario@ip del servidor [images/conexionssh/conexion exitosa]

**Reconocimiento del sistema
Una vez ingresado por via ssh a Ubuntu Server en Windows 11, se hace un reconocimiento inicial del sistema, aplicando comandos como
-Whoami
-ip a
-hostname
-pwd
Esto para identificar el sistema internamente, aunque la informacion es poca, muestra el usuario y muestra la ip, por que se crearan carpetas y archivos para ponder a prueba las habilidades que se han estudiado y analizado

**Creacion de una carpeta
En el servidor se ha creado una carpeta llamada lab_prueba y en la maquina cliente se ha creado de igual manera una carpeta ambos en diferentes usuarios, como admin y el principal 
Cuando se crea la carpeta de prueba, se corrobora en el servidor tanto la carpeta creada y el archivo de texto creado dentro de esa carpeta
Esto demuestra que la conexion por medio local con el router, configuracion en los archivos netplan y dhcp config se aplicaron correctamente [images/conexionssh/visualizacion de archivos en el servidor]

**Creacion de un archivo
Realizada la conexion ssh de una maquina con Windows 11 y servicio ssh con conexion remota, se procedio a crear un archivo txt, para que en el servidor se pueda observar la informacion que contiene dicho archivo txt [images/conexionssh/archivo maquina cliente]

**Creacion de grupos
Para continuar con la practica, se crearon 2 grupos y 3 usuarios
contabilidad e it
con usuarios conta1, conta2 y ti1
Cada uno de los usuarios tienen nombres ficticios
Con ayuda de los comandos usermod -aG grupo usuario
se cambio el grupo al que pertenecian los usuarios, manteniendo mas extructura en los grupos [images/conexionssh/grupos en el servidor]


**Visualizacion de usuarios en grupos
Para el caso de visualizar que usuarios estan en un grupo, se instalo el comando members para la visualizacion de los usuarios en un grupo
con el comando members nombre del grupo
para visualizar que usuarios pertenecen a ese grupo de manera mas sencilla [images/conexionssh/visualizacion de grupos y usuarios]

Conclusiones
Para este proyecto sencillo pero inicial, me ha ayudado a recordar las bases de los comandos para la ejecucion correcta del servidor
desde cambiar las IP de archivos editables para una correcta comunicacion local, hasta añadir usuarios a un grupo especifico, no solo eso sino tambien el uso de comandos instalables para observar informacion de manera mas concreta y entendible
asi como la ubicacion de el repositorio correcto para la creacion de archivos y carpetas, usar comandos como chmod, addgroup, adduser y otros mas como cat /etc/group para observar los grupos creados y mover a un usuario a un grupo especifico
Como mencion honorifica, poder realizar configuraciones en virtualBox para reconocer la conexion LAN y mas importante lograr una conexion remota con ayuda de ssh.
